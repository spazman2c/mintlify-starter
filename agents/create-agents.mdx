# Create and Manage Agents

Use **Agents** to create specialized support agents for different products, docs areas, customer segments, or API domains.

## Create an Agent

<Steps>
  <Step title="Open Agents">
    Go to **Agents** in the workspace sidebar.
  </Step>
  <Step title="Create a new agent">
    Add a name and description that operators can recognize.
  </Step>
  <Step title="Attach context">
    Choose one or more sources from Context.
  </Step>
  <Step title="Tune behavior">
    Review persona, instructions, greeting, confidence threshold, and guardrails.
  </Step>
  <Step title="Save and test">
    Ask representative support questions before routing real customers to the agent.
  </Step>
</Steps>

## Agent Sections

| Section | Use it for |
| --- | --- |
| Overview | Summary, status, widget key, context intelligence, and high-level controls. |
| Context | Source attachments and coverage. |
| Behavior | Persona, greeting, runbook instructions, and response style. |
| Guardrails | Safety rules, confidence, handoff behavior, and boundaries. |

## Default Agent

Every workspace should have a default agent. If the current default is deleted, Woes promotes the oldest remaining agent to default.

## Widget Keys

Each agent can have its own widget public key. Use this when you want a different agent on each site, docs area, or product.

Find all agent keys in **Settings → Keys**.
