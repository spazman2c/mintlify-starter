# How to Use This Knowledge Base

Use this site as an operational reference first. The raw KT bundle may contain recordings, screenshots, and source artifacts, but those materials are intentionally not published here because this site cannot be assumed to be password protected.

## Search strategy

- Search by **command** when you know a failing job, such as `from-sps so`, `to-hubspot spupdate`, or `to-bestbuy resendasn`.
- Search by **integration or partner protocol** when the flow is unclear, such as `SPS`, `HubSpot`, `Channable`, or `Symphony`.
- Search by **table or config section** when debugging data, such as `integration_settings`, `iqconnect_syncflags`, `iqconnect_so`, or `iqconnect_batch_soupdate_resend`.
- Search by **component** when you need implementation context, such as `IqSync`, `CheatSheet`, `IqReader`, `VssLastMile`, or `IqConnectWebhooksAsync`.

## Reading order for incidents

1. Start with the relevant integration page.
2. Check the command reference for the expected argument pattern.
3. Review the support runbook for the failure type.
4. Use developer reference pages when the runbook does not explain the behavior.
5. Go back to private source storage only when you need raw transcripts, screenshots, or customer-specific evidence.
