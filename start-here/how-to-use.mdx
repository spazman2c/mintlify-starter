# How to Use This Knowledge Base

Use this site as an operational reference first and an archive second. The pages are organized so the cleanest explanations appear before the raw meeting transcripts and source artifacts.

## Search strategy

- Search by **command** when you know a failing job, such as `from-sps so`, `to-hubspot spupdate`, or `to-bestbuy resendasn`.
- Search by **customer or partner** when the flow is unclear, such as `OSI`, `Renewtech`, `PARTpoint`, `Best Buy`, or `HubSpot`.
- Search by **table or config section** when debugging data, such as `integration_settings`, `iqconnect_syncflags`, `configsection.hubspot`, or `query.grossmargin`.
- Search by **artifact name** when you need original material, such as `HubSpotWebJobs`, `Publish-LKG`, or `Material List Tutorial`.

## Reading order for incidents

1. Start with the relevant integration page.
2. Check the command reference for the expected argument pattern.
3. Review the support runbook for the failure type.
4. Use transcript search only if the runbook does not explain the case.
5. Use source inventory when you need implementation-level context.
