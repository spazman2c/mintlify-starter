# Woes Documentation

Woes is an API-native developer support platform. It brings live chat, email, Discord, operator inboxes, API documentation ingestion, authenticated API testing, and grounded AI support agents into one workspace.

The core idea is simple: customers ask questions from any supported channel, and Woes answers from your actual API context. When the agent does not have enough evidence, it asks a focused question or hands the conversation to a human operator.

## What Woes Helps You Do

| Area | What you can do |
| --- | --- |
| Channels | Unify live chat, email, and Discord into one conversation model. |
| API context | Scan docs and source repositories into endpoint-level support context. |
| AI agents | Answer developer questions with citations, confidence thresholds, clarification, and handoff. |
| Operators | Triage conversations, take over AI-handled threads, create issues, add notes, use macros, and coordinate work. |
| Customers | Group conversations into client/account profiles and inspect customer context. |
| Analytics | Review operational metrics, agent quality, surveys, feedback, sources, and client activity. |
| Settings | Manage users, roles, channels, tags, statuses, macros, automations, keys, surveys, notifications, and install snippets. |

## Recommended First Path

<Steps>
  <Step title="Add API context">
    Start with [Add API Context](/quickstart/add-context). A good first source is your OpenAPI file, public docs URL, Postman documenter URL, or a GitHub repository containing docs and SDK examples.
  </Step>
  <Step title="Configure your first agent">
    Open [Agents](/agents/overview), attach the source, and let Woes generate a context-aware operating brief.
  </Step>
  <Step title="Install the widget">
    Use the agent's widget key in your install snippet. See [Install the Widget](/quickstart/install-widget).
  </Step>
  <Step title="Run support from the inbox">
    Monitor conversations in [Inbox](/platform/inbox), take over when needed, and convert customer work into [Issues](/platform/issues).
  </Step>
  <Step title="Measure and improve">
    Use [Analytics](/platform/analytics), [Surveys](/settings/surveys), and message feedback to improve the agent and support process.
  </Step>
</Steps>

<Note>
  Woes is designed around developer-support accuracy. The agent should never invent endpoints, auth rules, schemas, response shapes, or production behavior.
</Note>
