# IQ Connect Knowledge Base

This Mintlify site is the clean operational layer for IQ Connect knowledge transfer. It keeps runbooks, architecture notes, command patterns, and integration behavior searchable without publishing raw transcripts, source archives, customer identifiers, tenant keys, or deployment secrets.

## What is covered

| Area | What to use it for |
| --- | --- |
| Start Here | Orientation, system map, and publishing/security rules. |
| Operations | Production command patterns, support triage, resend procedures, and deployment guidance. |
| Integrations | Partner-specific behavior for fulfillment, CRM, catalog, and marketplace flows. |
| Developer Reference | Code-level concepts, component responsibilities, and database tables. |

## Recommended path

1. Read [System Map](/start-here/system-map) for the architecture.
2. Read [Security Boundary](/start-here/security-boundary) before publishing, sharing, or adding source material.
3. Use [Command Reference](/operations/command-reference) for job syntax.
4. Use the integration pages and [Support Runbook](/operations/support-runbook) during incidents.

## Publication rule

This site is written as if it may be visible without password protection. Do not add raw meeting transcripts, customer names, tenant identifiers, API keys, publish settings, screenshots containing customer data, or exact production file names. Use placeholders such as `<tenant-key>`, `<customer-alias>`, `<app-service-name>`, and `<webjob-folder>` instead.
