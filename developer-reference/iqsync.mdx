# IqSync

`IqSync` is the main orchestrator for IQ Connect data flows. It is a .NET console application deployed as Azure WebJobs. Command-line arguments identify the integration, feed, tenant context, environment, and sync mode.

## Command-Line Interface

```text
IqSync <integrationName> <feedType> [tenantID] [production|test] [reset|delta]
```

| Arg | Purpose | Examples |
| --- | --- | --- |
| `integrationName` | Which integration to invoke. | `from-bestbuy`, `to-bestbuy`, `to-hubspot`, `from-sps`, `to-channable` |
| `feedType` | Which data feed to process. | `itemsku`, `bom`, `subs`, `avl`, `price`, `so`, `shipment`, `invoice`, `returns`, `sp`, `catalog` |
| `tenantID` | Tenant identifier used to resolve private API/configuration context. | `<tenant-key>` or `<customer-alias>` |
| `mode` | Live or test mode. | `production`, `test` |
| `reset` | Full reload or incremental run. | `reset`, `delta` |

## Execution Flow

```mermaid
flowchart LR
  Args["Parse args"] --> CheatSheet["Populate CheatSheet"]
  CheatSheet --> Tenant["Validate tenant"]
  Tenant --> Handler["Dispatch handler"]
  Handler --> PostOp["Post-operation update"]
```

The `Main()` method switches over `integrationName`, populates `CheatSheet`, validates tenant context, and delegates to a feed handler.

| Handler | Direction | Used By |
| --- | --- | --- |
| `GetDataFromVendor()` | Reads from IQ API, writes to external system. | `to-bestbuy`, `to-sps`, `to-staples`, `to-ebay` |
| `PostDataToVendor()` | Reads from external system, writes to IQ API. | `from-bestbuy`, `from-sps`, `from-sellercloud` |
| `GetCrmDataFromIqr()` | Reads from IQ API, pushes to CRM. | `to-hubspot` feeds |
| `GetDataFromVendor_Channable()` | Reads IQ catalog, formats for Channable. | `to-channable` |
| `GetDataFromVendorInLoop()` | Processes per-SO ASN/invoice loops. | `to-bestbuy`, `to-sps` shipment and invoice feeds |

## Override and Resend Mechanism

When override mode is enabled, `IqSync` reads SO IDs from resend queue tables instead of scanning for unprocessed records.

| Override Feed | Source Table | Purpose |
| --- | --- | --- |
| `resendasn` | ASN resend queue. | Re-transmit a selected shipment. |
| `resendinvoice` | Invoice resend queue. | Re-transmit a selected invoice. |
| `resendsoupdate` | SO update resend queue. | Re-transmit a selected order update. |

After each resend, the processed record is deleted from the resend queue to prevent duplicates.
