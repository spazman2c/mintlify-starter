# Developer Reference Overview

IQ Connect synchronizes IQ Reseller with external trading partners, CRMs, and marketplaces. It runs primarily in Azure using scheduled WebJobs, Azure Functions, SQL tables, and SFTP/API handoffs.

## Solution Shape

| Area | Notes |
| --- | --- |
| Core engine | `IqSync`, `CheatSheet`, `IqReader`, and IQ Connect database tables. |
| Integration wrappers | Symphony, SPS Commerce, HubSpot, Channable, SellerCloud, and shell integrations. |
| Azure Functions | Webhook receivers and async quote-to-order orchestration. |
| Support services | `VssLastMile`, `IqFeedApi`, helper libraries, and web/dashboard projects. |

## Architecture Pattern

The hub is the `IqSync` console application, deployed as Azure WebJobs. Each spoke is an integration-specific wrapper class that handles protocol translation, feed formatting, SFTP/API transport, and post-operation state tracking.

## Active Integration Categories

| Category | Description |
| --- | --- |
| Symphony fulfillment | XML/SFTP workflow for catalog, orders, ASN, invoices, returns, cancellations, and ETA updates. |
| SPS Commerce | EDI-style order, ASN, invoice, and acknowledgement workflow. |
| HubSpot | CRM sync for companies, contacts, deals, line items, gross margin, and quote-to-order webhooks. |
| Channable | Product catalog feed for marketplace distribution. |

## Data Flow Directions

| Command Prefix | Direction | Example |
| --- | --- | --- |
| `from-bestbuy` | External partner to IQ Reseller | Inbound purchase orders, returns, and cancellations. |
| `to-bestbuy` | IQ Reseller to external partner | Catalog, ASN, invoices, and ETA updates. |
| `to-hubspot` | IQ Reseller to HubSpot | CRM sync of deals, companies, and contacts. |
| `to-channable` | IQ Reseller to Channable | Product catalog feed. |
| `from-sps` / `to-sps` | Bidirectional with SPS Commerce | Inbound POs, outbound ACK, ASN, and invoices. |

## Key Design Principles

- Jobs run unattended through schedules, manual WebJob execution, or webhook-triggered Functions.
- `IqSync` is lightweight and can be reused across many feed-specific WebJobs.
- Tenant context is resolved at runtime from command arguments and private configuration.
- Delta mode handles recurring syncs; reset mode is reserved for controlled full reloads.
