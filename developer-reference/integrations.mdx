# Integrations

Each external system integration is encapsulated in a dedicated wrapper class within `IqSync/ApiWrapper`. These classes handle protocol translation, feed formatting, SFTP/API file management, and post-operation state tracking.

## Active Integrations

| Integration | Main Files | Protocol | Feeds |
| --- | --- | --- | --- |
| Symphony | `Symphony.cs` | SFTP / XML | Catalog, orders, ASN, invoices, returns, cancellations, ETA updates. |
| SPS Commerce | `SpsCommerce.cs`, `SpsCommerceDS.cs`, `SpsFormatConverter.cs` | SFTP / XML / JSON | Orders, ASN, invoices, acknowledgements. |
| HubSpot | `HubSpot.cs` | REST API / JSON / Webhooks | Companies, contacts, deals, line items, gross margin, quote-to-order. |
| Channable | `Channable.cs` | REST API / feed output | Full product catalog: SKU, brand, condition, pricing, stock, categories. |

## Common Integration Pattern

1. Constructor receives `IqReader`, `feedType`, and `CheatSheet`.
2. Feed-specific preparation transforms IQ XML or query results into the target format.
3. Writer method stages the file, posts the API payload, or sends through SFTP.
4. `PostOpUpdateQuery` is prepared to mark records as processed.
5. `IqSync` executes the post-operation update through `VssLastMile`.
