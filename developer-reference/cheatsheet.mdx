# CheatSheet

`CheatSheet` is the central runtime configuration object for an `IqSync` execution. It is constructed from app configuration, command-line arguments, tenant-specific settings, and database-backed integration settings. Every wrapper receives a `CheatSheet` reference.

## Nested Configuration Sections

| Section Class | Purpose | Key Properties |
| --- | --- | --- |
| `ActiveSection` | Per-integration settings such as SFTP host, folders, credentials, feed filters, and condition codes. | `Host`, `Port`, `Cred1`, `Cred2`, `InputFolder`, `OutputFolder`, `SshPrivateKeyFile`, `FeedFilters` |
| `AzureSection` | Azure WebJob storage settings. | `AzureWebJobsDashboard`, `AzureWebJobsStorage`, `WebjobsHistorySize` |
| `IqApiSection` | IQ Reseller API endpoints and URI templates. | `SigninUrl`, `EndPointUrl`, `ApiToken`, `Uri_*` properties |
| `IqTenantSection` | Resolved tenant context. | `ConnectionString`, `TenantId`, `TenantAlias`, `DatabaseName` |

## Configuration Loading Strategy

| Constructor | Used By |
| --- | --- |
| App settings, active section, connection strings, and production mode. | Simple integrations that do not need tenant context. |
| Same as above plus tenant API key. | Tenant-aware fulfillment and EDI integrations. |
| Same as above plus CRM integration flag. | CRM integrations that load settings from the database. |

The CRM-oriented constructor calls `ReadActiveIntegrationSettingsFromDb()`, queries `integration_settings`, and parses XML or JSON into a name/value collection. This allows per-tenant integration configuration without redeploying the WebJob.

## Encrypted URL Parameters

The VssLastMile query endpoint can receive an encrypted connection-string parameter. `EncryptionHelper.GetEncryptedParameterWithUrlEncoding()` encrypts the parameter before it is placed in URLs or logs.
