# Azure

IQ Connect runs across Azure App Services, WebJobs, Azure Functions, and Azure SQL. The design keeps most traffic inside Azure; the external handoff is usually the final SFTP transfer to a partner mailbox.

## Component Map

| Component | Azure Service | Details |
| --- | --- | --- |
| `IqSync` | Triggered WebJobs | One executable is reused across feed-specific WebJobs. Commands identify direction, feed, tenant, and mode. |
| `IqConnectCore` | App Service | Admin/dashboard app. Use `<app-service-name>` in published docs. |
| `VssLastMile` | App Service | Provides query and marshal endpoints used by integration jobs. |
| `IqConnectWebhooks` | Azure Functions | HTTP-triggered webhook endpoints. |
| `IqConnectWebhooksAsync` | Durable Functions | Multi-step quote-to-IQ-SO orchestration. |
| `SymphonyFunctionApp` | Azure Functions | Timer-triggered Symphony support tasks. |
| Tenant databases | Azure SQL | Per-tenant IQ Reseller databases. |
| `IqFeedApi` | App Service | REST API for feed access. |

## WebJob Naming

WebJobs generally follow this pattern:

```text
{integration}-{customer-alias}-{sequence}-{feedtype}
```

Examples should stay generic in published docs:

| WebJob | Meaning |
| --- | --- |
| `sym-<alias>-01-itemsku` | Symphony item catalog. |
| `sym-<alias>-06-so` | Inbound vendor orders. |
| `sym-<alias>-07-so-updates` | SO updates / ETA updates. |
| `sym-<alias>-08-shipment` | ASN. |
| `sym-<alias>-11-invoice` | Invoice. |

## Kudu Usage

Use Kudu when a support runbook requires running an executable manually:

```text
Azure Portal -> <app-service-name> -> Development Tools -> Advanced Tools -> Go -> Debug console -> CMD
```

For promise-date recovery, see [Promise Date Resend](/operations/best-buy-promise-date-resend).
