# Publish and Deployment

Deployment material must stay separate from the published docs. Publish profiles, screenshots, Kudu paths, and exact App Service names can expose production structure or credentials.

## Safe Deployment Notes

| Topic | Guidance |
| --- | --- |
| App Service | Refer to `<app-service-name>` unless the reader is already inside the private Azure portal. |
| Kudu | Document the navigation path, not account-specific URLs or screenshots. |
| Publish profiles | Do not publish `.PublishSettings` files or their filenames. They may contain sensitive deployment metadata. |
| Screenshots | Avoid publishing screenshots of Azure, SQL, SFTP, or Kudu unless they are redacted. |
| WebJob paths | Use placeholders such as `<webjob-folder>` unless the exact path is required inside a private runbook. |

## Kudu Pattern

```text
Azure Portal -> <app-service-name> -> Development Tools -> Advanced Tools -> Go -> Debug console -> CMD
```

From Kudu CMD, locate an executable when needed:

```cmd
cd /d D:\home\site\wwwroot
dir /s /b IqSync.exe
```

## If Credentials Were Exposed

Rotate the credential, remove the file from the current tree, and consider repository history cleanup if the repository was public or broadly accessible.
