# Command Reference

IQ Connect commands use a consistent shape:

```text
<direction> <operation> <tenant-or-customer> <environment> [mode]
```

Use real production values only inside the Azure job or private support session. Published docs should use placeholders.

## Direction Conventions

| Prefix | Meaning |
| --- | --- |
| `from-*` | Pulls partner data into IQ Reseller, such as orders, returns, cancellations, or local replay files. |
| `to-*` | Sends IQ Reseller data outward, such as catalog, inventory, ASN, invoices, CRM data, or order updates. |
| `bestbuy service` | Service-style Symphony workflow command. |

## Symphony / Best Buy

| Purpose | Command pattern |
| --- | --- |
| Import orders | `from-bestbuy so <tenant-key> production` |
| Import orders, delta mode | `from-bestbuy so <tenant-key> production delta` |
| Send SO updates / ETA updates | `to-bestbuy soupdate <tenant-key> production delta` |
| Manually resend SO updates / ETA updates | `to-bestbuy resendsoupdate <tenant-key> production delta` |
| Resend ASN | `to-bestbuy resendasn <tenant-key> production delta` |
| Send availability | `to-bestbuy avl <tenant-key> production delta` |

For the Kudu command used to force missing promise date updates, see [Promise Date Resend](/operations/best-buy-promise-date-resend).

## Execution Flow

1. Resolve the active integration section from the command arguments.
2. Populate `CheatSheet` with integration, Azure, IQ API, and tenant settings.
3. Validate the tenant context.
4. Read source records through `IqReader` or a CRM-specific reader.
5. Format the partner-specific payload.
6. Send or stage the payload.
7. Update sync state and feed history.
