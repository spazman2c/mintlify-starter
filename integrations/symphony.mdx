## Best Buy Symphony Integration 

Full bidirectional EDI integration with Best Buy's Symphony platform. Used by PARTpoint . Handles the complete vendor lifecycle: catalog data, purchase orders, shipment confirmations (ASN), invoices, returns, and cancellations. All data exchange is via XML files over SFTP.

### SFTP Configuration 

Symphony files are exchanged via Best Buy's managed file transfer (MFT) SFTP server. The Uhaul class handles file staging and transfer, routing through the VssLastMile service for Azure-to-SFTP bridging. 

 | Setting | Production 
 | Host | mft-tst.bestbuy.com (production varies by environment) 
 | Auth | SSH private key ( CertKey/iqbb_pp_openssh ) 
 | Input Folder | /from_bby (inbound POs, returns, cancels) 
 | Output Folder | /to_bby (outbound catalog, ASN, invoices) 
 | File Prefix | Ppnt (PARTpoint) 

### Key Behaviors 

Partial void handling: The _ConstPartialVoid = "BBYVOID" marker is used when a line item needs to be voided from an order without cancelling the entire SO. The wrapper generates appropriate void elements in the outbound XML. 

Invalid line number filtering: Records with line number 99999 (constant k_InvalidLineNumber ) are treated as bad records and excluded from ASN feeds. This prevents malformed line items from reaching Best Buy. 

Feed history cleanup: The CleanupDatabase() method in IqSync purges catalog feed content older than 3 days and order feed content older than 42 days from iqconnect_feedhistory to manage database size.
