# Best Buy Symphony

Best Buy Symphony is a fulfillment integration over XML files and SFTP. It covers the full order lifecycle from catalog publication through PO import, acknowledgements, ETA updates, ASN, invoices, returns, and cancellations.

## Supported Feeds

| Feed | Direction | Description |
| --- | --- | --- |
| `itemsku` | IQ to partner | Master item catalog: SKU, manufacturer, description, UPC, and related item data. |
| `bom` | IQ to partner | Part associations / bill of materials. |
| `subs` | IQ to partner | Item substitutions. |
| `avl` | IQ to partner | Inventory availability by quantity and condition. |
| `price` | IQ to partner | Pricing updates. |
| `so` | Partner to IQ | Vendor orders / purchase orders. |
| `soupdate` | IQ to partner | Order acknowledgements, ETA updates, and vendor order updates. |
| `shipment` | IQ to partner | ASN shipment notices with tracking. |
| `invoice` | IQ to partner | Vendor invoices. |
| `returns` | Partner to IQ | Return requests. |
| `cancel` | Partner to IQ | Cancellation requests. |
| `returnack` | IQ to partner | Return and cancellation acknowledgements. |

## Key Components

| Component | Purpose |
| --- | --- |
| `IqSync/ApiWrapper/Symphony.cs` | Main wrapper. Builds XML, manages feed-specific logic, and writes outbound files. |
| `SymphonyReturnCancel.cs` | Return/cancel acknowledgement data structures. |
| `Uhaul` | Handles file staging and transfer between Azure and SFTP paths. |
| `iqconnect_so` | Tracks partner PO to IQ SO mapping, workflow status, feed GUID, and last ETA sent. |
| `iqconnect_posted_voupdates` | Tracks posted vendor order update statuses. |
| `iqconnect_batch_soupdate_resend` | Manual resend list for SO update / ETA recovery. |

## SFTP Configuration

| Setting | Value |
| --- | --- |
| Input folder | Inbound PO, return, and cancellation folder configured per tenant. |
| Output folder | Outbound catalog, ASN, invoice, and update folder configured per tenant. |
| File prefix | Tenant-specific prefix. Do not publish production prefixes. |
| Transport | SFTP via managed file transfer. |

## Promise Date / ETA Updates

The customer updates the SO promise date in IQ Reseller. IQ Connect reads that date from `st.deliverydate` and sends it in the `voupdates` file as `ExpectedActivityDate`.

For manual recovery when promise dates were entered but the partner did not receive the update, use [Promise Date Resend](/operations/best-buy-promise-date-resend).

## Common Commands

```bash
from-bestbuy so <tenant-key> production
from-bestbuy so <tenant-key> production delta

to-bestbuy soupdate <tenant-key> production delta
to-bestbuy resendsoupdate <tenant-key> production delta
to-bestbuy resendasn <tenant-key> production delta
to-bestbuy avl <tenant-key> production delta
```

## Operational Notes

| Behavior | Detail |
| --- | --- |
| Partial voids | The `BBYVOID` marker is used when a line item is voided without cancelling the full SO. |
| Invalid line numbers | Line number `99999` is treated as malformed and filtered from ASN feeds. |
| Feed history cleanup | Catalog feed content is purged quickly; order feed content is retained longer for support lookback. |
