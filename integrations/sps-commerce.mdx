## SPS Commerce Integration 

EDI integration via SPS Commerce, used by OSI Global . Added November 2024. Structurally mirrors the Symphony integration pattern (SFTP-based XML) but with SPS-specific formatting and document types. Supports the full order-to-invoice lifecycle.

### Local Development Support 

When feedType == k_solocal , the wrapper reads from local file system directories instead of SFTP — useful for development and debugging SPS document formats without connecting to the live SFTP server.
