# SPS Commerce

SPS Commerce is an EDI fulfillment integration. It follows the same general IQ Connect pattern as Symphony, but uses SPS-specific XML formats and document rules.

## Supported Workflow

| Direction | Command | Purpose |
| --- | --- | --- |
| SPS to IQ | `from-sps so <customer-alias> production` | Import purchase orders and create IQ sales orders. |
| SPS to IQ | `from-sps solocal <customer-alias> production delta` | Replay or test local SPS order payloads. |
| IQ to SPS | `to-sps shipment <customer-alias> production delta` | Send ASN / shipment notices. |
| IQ to SPS | `to-sps invoice <customer-alias> production delta` | Send invoices. |
| IQ to SPS | `to-sps resendasn <customer-alias> production delta` | Resend selected shipment notices. |
| IQ to SPS | `to-sps resendinvoice <customer-alias> production delta` | Resend selected invoices. |
| IQ to SPS | `to-sps resendsoupdate <customer-alias> production delta` | Resend selected order updates. |

## Local Development

When the feed type is `solocal`, the wrapper reads test payloads from local folders instead of SFTP. Use this path to debug SPS document formatting without touching the live partner mailbox.

## Troubleshooting

| Symptom | First Checks |
| --- | --- |
| Order did not post | Check the inbound file, local replay path if used, IQ API response, and mirror copy. |
| ASN did not send | Check the unshipped SO list, line-number mapping, shipment data, and generated XML. |
| Invoice did not send | Check the invoice eligibility query, address length fields, and invoice resend list. |
| Line number mismatch | Compare partner PO line numbers against IQ SO detail internal comments. |
